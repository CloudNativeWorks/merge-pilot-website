Key Features

Everything you need to streamline merge request management across your organization.

Automated Classification

Branch pattern matching to automatically classify MRs as hotfix, feature, or release — with ordered rule evaluation and first-match-wins logic.

Jira CMG Integration

Automatic Change Management Group ticket creation with template support, custom field mapping, and bidirectional sync between GitLab and Jira.

Risk Assessment

Multi-factor risk scoring based on change windows, service criticality, diff size, and blackout periods — ensuring safe deployments every time.

Secret Scanning

Gitleaks-powered secret detection with severity-based alerting, allowlist support, and automated blocking of MRs containing exposed credentials.

Multi-Tenant

Company-level isolation with separate GitLab, Jira, and LDAP configurations. Each tenant gets independent classification rules and project mappings.

Approval Workflow

Jira-driven approval and rejection with auto-approve capabilities, approval fatigue detection, and full audit trail of every decision.

Classification

Smart Classification Engine

  • Regex-based branch pattern matching — define rules like hotfix/*, feature/*, release/*
  • Ordered rule evaluation — first match wins, giving you full control over priority
  • Auto-approve for hotfix/emergency — skip CMG for critical fixes
  • Automatic assignment — priority level, change type, and risk level set per rule
classification_rules 
{
  "pattern": "^hotfix/.*",
  "change_type": "emergency",
  "risk_level": "high",
  "auto_approve": true,
  "priority": 1
}
{
  "pattern": "^feature/.*",
  "change_type": "standard",
  "risk_level": "medium",
  "auto_approve": false,
  "priority": 2
}
